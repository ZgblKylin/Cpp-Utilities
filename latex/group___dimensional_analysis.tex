\doxysubsection{Detailed Description}
Helper classes, typedefs and functions for dimensional analyse. 

Provide helperful functionalities for dimensional analysis with pros\+:~\newline
 {\bfseries{Zero-\/cost abstraction}} Using template to encapsulate functionalities, grants same memory-\/layout and runtime performance as primitive types.~\newline
 {\bfseries{Strong typed}} All values in dimensional objects has its unique unit$\ast$$\ast$, calculation between them is guaranteed by compiler warning and {\bfseries{error}}.~\newline
 {\bfseries{Non-\/standard unit}} of same \char`\"{}type\char`\"{}, like {\ttfamily yard} vs {\ttfamily meter}, is supported, and will be regarded like {\bfseries{same}} unit, {\ttfamily std\+::ratio} is used for modifier of the unit. Calculations between same unit with different ratio is safe, ratios of each operand will be performed into the calculation.~\newline
 {\bfseries{Add/subtract}} with {\bfseries{same}} unit is allowed, whereas with different units will cause compile error.~\newline
 {\bfseries{Multiply/divide}} values with {\bfseries{any}} units is safe, a new value with {\bfseries{new unit}} will be generated.~\newline
 {\bfseries{User defined unit}} is supported. Helper types is provided to derive new unit from existsing units.~\newline
